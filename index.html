Windows 下搭建免费个人博客

发表于 2016-07-21

最近在简书上看到一篇关于创建个人博客的文章 – 5分钟 搭建免费个人博客，于是欣喜若狂，说干就干(嘿嘿，对于我的实干精神还是颇为满意滴。。。)可博主5分钟搞定的事情，15分钟后的我仍一头雾水（心塞中。。。），于是记录下自己的辛酸路程，以便其他的小伙伴参考。(我可是一个乐于助人的小姑凉~~~)

工具/原料

Nodejs && Git for windows

在下载git的过程中可能会遇到下载失败的情况，此时可以在别处寻求下载资源.

方法/步骤

1. 创建Github域名和空间

1.1 注册

首先，你需要注册一个Github账号

1.2 创建仓库

当拥有一个Github账号之后，就可以创建一个仓库（repository）来存储我们即将创建的博客网站，点击首页任意位置出现的New Repository按钮创建仓库,Respository name 中的username.github.io 的username 一定与前面的Owner 一致，记住你的username下面会用到。

第一步就这样完成了,是不是很简单啊!

2. 安装

Hexo是基于Nodejs的博客框架,安装Hexo之前必须安装Git和Nodejs.

2.1 安装Git(Git for windows)

直接点击Git for windows或下载资源便可进行下载安装.

测试是否成功安装Git,选中任意文件夹点击右键,如出现下图,则说明Git安装成功.

2.2 安装Nodejs

点击Nodejs进行下载安装,参看Nodejs安装图解教程(嘿嘿~~ 下一步 + 下一步 毫无压力哦!)

2.3 安装Hexo

Git 和 Nodejs安装成功之后,Hexo的安装也就水到渠成了.

$ npm install hexo-cli -g

3. 编写博客及发布

接下来需要用Hexo初始化一个博客,然后更改一些自定义的配置,或者加上自己喜欢的主题,写上第一篇文章,然后发布到自己的个人Github网站上(username.github.io)(嘿嘿~~ 内心是不是有点小激动呢!).

3.1 创建博客

新建一个名为blog(只是举个例子啦,当然你有绝对的自由)的文件夹,点击右键选择Git Bash Here,会弹出如下的命令行弹框.

键入如下的命令:

$ hexo init ijobs.github.io

执行成功后,在blog文件夹下会创建出一个名为ijobs.github.io文件夹(当然啦!Hexo的安装命令也是在上面的命令行弹框中执行的)

3.2 更改配置

主题的安装

对于主题你可以选择任意喜欢的,这里以安装next主题为例,当然Hexo也有更多的主题供你选择.

$ cd ijobs.github.io $ git clone https://github.com/iissnan/hexo-theme-next themes/next
设置详情(嘿嘿~~ 此处有Hexo的官方设置文档,非常详尽哦!)

基础配置:打开文件位置ijobs.github.io/_config.yml修改相应的键值对即可:

3.3 写文章

所有的基础框架都已经创建完成,接下来就可以开始写你的第一篇博客了,在ijobs.github.io/source/_posts下创建你的第一个博客吧!

例如,你可以创建一个名为FirstBlog.md的文件(可能你现在需要学习一些MarkDown的基本语法了,但不用怕,真的很好上手哦!)

3.4 测试

$ hexo s

测试服务启动,你可以在浏览器中输入 https://localhost:4000 进行访问了.

当如下页面跳出来的那一刻是不是很欣喜呢!

3.4 安装[hexo-deployer-git]自动部署发布工具

$ npm install hexo-deployer-git --save

(哎!在这里没少耽搁时间,不过功夫不负有心人,最终安装成功,如果由于网速或其他什么原因第一次未能安装成功,不用怕,重复执行上面的代码,总会成功的!)

3.5 发布

测试成功后,就可以生成静态网页文件发布至我们的Github pages中.

$ hexo clean && hexo g && hexo d

此时终端会让你输入Github的邮箱和密码,正确输入后,稍等片刻,就会把你的博客上传至Github了.以后每次写完博客,执行上面的命令就可以直接发布了,是不是非常的方便啊!

前端分页和后端分页的优缺点及相关实现

发表于 2016-07-21

鉴于Handlebars.js前端模板的使用,考虑到是不是可以由前端直接实现分页效果,但由于不太清楚前端和后端实现分页的具体区别和优缺点,所以就在知乎上寻求大神们的帮助.

在论述前端分页和后端分页的优缺点之前,先简单介绍一下Handlebars.js.
Handlebars是Javascript一个语义模板库,通过对view和data分离来快速构建web模板.它采用"Logic-less template"(无逻辑模板)的思路,在加载时被预编译,而不是到了客户端执行到代码时再去编译.这样可以保证模板加载和运行的速度.
Js模板引擎handlebars.js实用教程

针对知乎大神们的回答整理了比较满意的答案:

前后分页的优缺点

  • 对于局部刷新的数据采用前端分页用户体验比较好,比如评论列表,点赞用户列表.

  • 对于数据量较大.要求seo的数据页采用后端分页.

  • 当然也可以两种模式相结合,比如微博列表加载数据,前10页是Ajax获取分页数据,后10页是后端分页,在前10页上的a标签上也加上src地址利于蜘蛛的抓取,但是禁用锚点链接功能改用click通过Ajax获取数据.

可能更好的实现方式

PS:分页即列表页先加载300条,但只显示100条,这样反而让分页逻辑混乱了,还要判断分页是不是已经加载过了,而且现在的趋势是”非必要不加载”,下拉更新.图片滚动到了才加载,只要检测到用户需要这部分数据才去加载,多省宽带,所以前端分页基本上没有可以一提的优点,反而让程序逻辑多绕几个圈.

以上的两个回答,个人感觉都比较认可,当然在项目中是采用前端分页还是后端分页,甚至是摈弃分页而采用”无必要不加载”的下拉更新都是要根据具体的情况来进行选择的.

前端图片优化技术汇总

发表于 2016-07-21

最近在知乎上看到一个关于网站图片优化的问题,看到一个叫秦墨鱼的回答,深深感受到前端知识的深度,也越来越感受到前端的魅力,现在将图片优化的一些技术记录如下:
(期待以后的自己也能够跟随前辈们的脚步,建成自己的前端知识库~~)

前端图片优化方案

(以淘宝网站来做案例分析)

1. 使用base64编码代替图片

场景：将图片转换为base64编码字符串inline到页面或css中，适用于图片大小小于2KB，页面上引用图片总数不多的情况。

2. 合并图片sprite(图片精灵)

场景：将多个页面上用到的背景图片合并成一个大的图片在页面中引用 ，可以有效的较少请求个数。但是合并的大图也不要太大。
比如淘宝首页的这个sprite图片

3. 图片延迟加载

场景：对应图片比较多的页面，可以考虑通过js来实现图片的延迟加载，先让一部分图片优先请求下来，当用户滚动页面的时候进一步加载图片。
我们看一下淘宝首页的实现，你打开chrome控制台，然后滚动页面，就会发现新的滚动就会导致大量图片的新请求

4. 使用css、svg、canvas或iconfont代替图片

场景：css方式可以用来绘制相对简单的团来代替图片，一般使用before或者after伪元素来丰富图案的复杂度。使用css,svg,iconfont减少图片尺寸，请求数据少。

5. 加载对应尺寸图片

场景：不同终端对同一个图片需求不一样，可以根据终端加载不同的图片来节省没必要的流量 。
查看淘宝首页图片的网络加载，会通过cdn技术，加上对应要加载的图片参数，比如60x60,加载需要的尺寸。

6. 预加载

场景：默认情况下浏览器会对页面中和当前域名（正在浏览网页的域名）不在同一个域的域名进行预获取，并且缓存结果，这就是隐式的DNS Prefetch。如果想对页面中没有出现的域进行预获取，那么就要使用显示的DNS Prefetch了，也就是使用link标签。
我们在查看淘宝网的前端代码可以看到前端加上了dns-prefetch，因为前端静态资源的域名和主域名http://taobao.com不一致

7. 更好的图片格式

场景：使用webp、bpg、sharpP等新图片格式来替代原始格式，这些具有更好的压缩比。
我们看淘宝这边，部分图片使用了webp格式，WEBP与JPG相比较，编码速度慢10倍，解码速度慢1.5倍，而绝大部分的网络应用中，图片都是静态文件。

发表于 2016-07-21

虽然已经有了很多优秀的Sass教程,但还是想记录一下自己学习Sass之初的点滴.

Sass的安装

1.安装ruby

因为sass依赖于ruby环境,所以在安装sass之前必须安装ruby.
详尽的安装教程

Window下安装ruby,直接下载rubyinstaller.exe安装就可以了.(安装非常简单,直接下一步+下一步就可以了!)

测试ruby是否安装成功:

ruby -v

执行上面的命令,如果可以看到ruby相应的版本号,说明安装成功.

2.安装sass

安装完ruby之后,在开始菜单中,找到刚刚安装的ruby,打开Command Prompt with ruby and Rails,如下图所示:

然后直接在命令行中输入下面的命令:

gem install sass

最后sass -v来检测sass是否安装成功.同样,如果看到sass相应的版本号,说明安装成功.

升级sass版本的命令如下:

gem update sass
